You can extend the functionality of this library easily with a basic knowledge of the AngularJS directives development and the Leaflet library API. The code is modular enough and easy to read, so feel free to add code or modify it to fit the library to your own needs. Furthermore, if you would like to merge your changes to the main development branch send a pull request via Github.

Building, testing and developing

Coding style: The style guide we try to use is the Airbnb Javascript Style Guide.

Build system: We use Grunt as a tool to automate building, linting, testing and minifying the code. You can see the development utilities used on the package.json file, and the grunt tasks on the Gruntfile.js file. Before we can use Grunt, we must install the required packages via npm, let's see an example of how to install the project dependencies. First, make sure you have npm and grunt-cli installed globally. See how to install the Grunt tools below.

Dependency system: We use Bower for the dependency resolution of libraries.

Using the Grunt tasks

Once you have the development dependencies installed, we can use our predefined grunt tasks. For example:

  • grunt test. Executes the karma unitary tests and the protractor e2e tests, reporting the actual state of the project.
  • grunt test:unit. Executes the unitary tests the karma framework.
  • grunt test:e2e. Executes the e2e tests of the protractor E2E framework.
  • grunt coverage. Generates a "coverage" folder with an istanbul report about the percentage of the code which is covered by the existing tests.
  • grunt default. The default task tries to build the library file and then runs the JSHint filter and unit tests.
