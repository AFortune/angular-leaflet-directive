How to collaborate

You can extend the functionality of this library easily with a basic knowledge of the AngularJS directives development and the Leaflet library API. The code is modular and easy to read so feel free to add code or modify it to fit the library to your own needs. Futhermore, if you would like to merge your changes to the main development branch send a pull request via github.

Building, testing and developing

Coding style: We use this .editorconfig file to define our coding style between IDE's. You can find more info about this on http://editorconfig.org/.

Grunt: We also use Grunt as a tool to automate building, linting, testing and minifying the code. You can see the development utilities used on the package.json file, and the grunt tasks on the Gruntfile.js file. Defined Grunt tasks

  • grunt test. Executes the unitary tests and the e2e tests of the karma framework.
  • grunt jshint. Lints the main JS files of the project (directive and tests) with the jshint utility.
  • grunt uglify. Minifies the library with UglifyJS.
  • grunt default. The default grunt task will start listening for project changes on our library file src/angular-leaflet-directive.js. Whenever it detects that the file has changed, it will execute the jshint task, the karma tests task, and the uglify task (if every previous task went ok) so the dist/angular-leaflet-directive.min.js file will be ready for production.
